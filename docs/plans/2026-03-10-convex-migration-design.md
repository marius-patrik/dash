# Convex Migration Design

## Summary

Migrate the Dash backend from Supabase + Fastify to Convex, and flatten the monorepo into a single app. Replace Supabase Auth with Clerk. Replace WebSocket streaming with Convex reactive queries.

## Motivation

- Simpler DX: eliminate separate server, auto-generated APIs
- Real-time built-in: reactive queries replace manual WebSocket management
- Full rewrite: app is evolving, want a better foundation
- Cost/hosting: no server to manage, Convex handles everything

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Migration strategy | Big bang rewrite | Small app, no production data, cleanest result |
| Auth provider | Clerk | Best Convex integration, built-in OAuth, free tier |
| Chat streaming | Convex actions + mutations | Fully native, no separate server |
| Project structure | Flat single app | Drop monorepo, simplest possible |
| Frontend bundler | Keep Rsbuild | Already working, Convex is bundler-agnostic |

## Schema

Maps existing 8 Supabase tables to Convex, plus a `streamingState` table for real-time chat.

```typescript
// convex/schema.ts
import { defineSchema, defineTable } from "convex/server";
import { v } from "convex/values";

export default defineSchema({
  users: defineTable({
    clerkId: v.string(),
    displayName: v.string(),
    avatarUrl: v.optional(v.string()),
    settings: v.optional(v.any()),
  }).index("by_clerk_id", ["clerkId"]),

  agentConfigs: defineTable({
    userId: v.id("users"),
    name: v.string(),
    model: v.string(),
    systemPrompt: v.string(),
    parameters: v.optional(v.object({
      temperature: v.optional(v.number()),
      maxTokens: v.optional(v.number()),
      maxTurns: v.optional(v.number()),
      maxBudgetUsd: v.optional(v.number()),
      effort: v.optional(v.string()),
    })),
    toolPermissions: v.optional(v.object({
      allowedTools: v.optional(v.array(v.string())),
      disallowedTools: v.optional(v.array(v.string())),
      permissionMode: v.optional(v.string()),
    })),
    isDefault: v.boolean(),
  }).index("by_user", ["userId"]),

  mcpServers: defineTable({
    userId: v.id("users"),
    name: v.string(),
    type: v.string(),
    command: v.string(),
    args: v.array(v.string()),
    envVars: v.optional(v.any()),
    status: v.string(),
  }).index("by_user", ["userId"]),

  skills: defineTable({
    userId: v.id("users"),
    name: v.string(),
    description: v.string(),
    content: v.string(),
    category: v.string(),
    version: v.number(),
    isTemplate: v.boolean(),
  }).index("by_user", ["userId"]),

  memories: defineTable({
    userId: v.id("users"),
    key: v.string(),
    value: v.string(),
    category: v.string(),
  }).index("by_user", ["userId"]),

  sessions: defineTable({
    userId: v.id("users"),
    agentConfigId: v.optional(v.id("agentConfigs")),
    agentSessionId: v.optional(v.string()),
    name: v.string(),
    status: v.string(),
    tags: v.array(v.string()),
    mcpServerIds: v.array(v.id("mcpServers")),
    skillIds: v.array(v.id("skills")),
    metadata: v.optional(v.any()),
  }).index("by_user", ["userId"]),

  messages: defineTable({
    sessionId: v.id("sessions"),
    role: v.string(),
    content: v.string(),
    toolCalls: v.optional(v.any()),
    toolResults: v.optional(v.any()),
    tokenCount: v.optional(v.number()),
    costUsd: v.optional(v.number()),
  }).index("by_session", ["sessionId"]),

  streamingState: defineTable({
    sessionId: v.id("sessions"),
    text: v.string(),
    activeTools: v.optional(v.any()),
    isStreaming: v.boolean(),
    error: v.optional(v.string()),
    costUsd: v.optional(v.number()),
  }).index("by_session", ["sessionId"]),

  contextPresets: defineTable({
    userId: v.id("users"),
    name: v.string(),
    description: v.optional(v.string()),
    includedMemories: v.array(v.id("memories")),
    includedFiles: v.array(v.string()),
    includedSkills: v.array(v.id("skills")),
  }).index("by_user", ["userId"]),
});
```

## Chat Streaming

Replaces WebSocket with Convex reactive queries:

1. Client calls `useMutation(api.sessions.sendMessage)` with `{ sessionId, content }`
2. Mutation saves user message, creates/resets `streamingState`, schedules agent action via `ctx.scheduler.runAfter(0, ...)`
3. Agent action (Node.js runtime) runs Claude SDK query
4. As streaming events arrive, action calls `ctx.runMutation()` to update `streamingState` (text, activeTools, isStreaming)
5. Client subscribes via `useQuery(api.sessions.getStreamingState, { sessionId })` for real-time updates
6. On completion, action saves final message to `messages` and deletes `streamingState`

Abort: client calls `cancelStream` mutation setting a `cancelled` flag. Action checks periodically and aborts.

Batching: action batches text updates (~100ms) to avoid excessive mutations.

### useChat hook

```typescript
function useChat(sessionId: Id<"sessions">) {
  const messages = useQuery(api.messages.bySession, { sessionId });
  const stream = useQuery(api.sessions.getStreamingState, { sessionId });
  const sendMessage = useMutation(api.sessions.sendMessage);
  const cancel = useMutation(api.sessions.cancelStream);

  return {
    messages,
    streamingText: stream?.text ?? "",
    activeTools: stream?.activeTools ?? {},
    isStreaming: stream?.isStreaming ?? false,
    error: stream?.error ?? null,
    costUsd: stream?.costUsd ?? null,
    sendMessage: (content: string) => sendMessage({ sessionId, content }),
    abort: () => cancel({ sessionId }),
  };
}
```

## Auth (Clerk)

- Clerk handles GitHub + Google OAuth + email/password
- `ConvexProviderWithClerk` wraps app, passes JWT automatically
- Every Convex function checks identity via `ctx.auth.getUserIdentity()`
- Clerk webhook syncs user data to `users` table
- Client uses `<Authenticated>` / `<Unauthenticated>` components

## Frontend Changes

### Stays the same
- All UI components (shadcn/ui, Tailwind CSS 4)
- Router (wouter) and route structure
- Page layout (sidebar, topbar, dashboard layout)
- All visual design / JSX

### Changes in every page
- `useEffect` + `apiGet` + `useState` -> `useQuery`
- `apiPost`/`apiPatch`/`apiDelete` -> `useMutation`
- Types from `@dash/shared` -> auto-generated from Convex schema

### Deleted
- `src/lib/api.ts`, `src/lib/ws.ts`, `src/lib/supabase.ts`
- `src/providers/auth-provider.tsx`

### New/modified
- `src/providers/convex-provider.tsx`
- `src/lib/hooks/use-chat.ts` (reactive queries)
- `src/lib/constants.ts` (AVAILABLE_MODELS, etc.)
- All page components (swap fetch for queries/mutations)
- `src/router.tsx` (ProtectedRoute -> Authenticated)
- `src/app/login/page.tsx` (Clerk SignIn)

## Deployment

- Convex: auto-deployed via `npx convex deploy`
- Vercel: continues serving static SPA
- Claude API key: set via `npx convex env set`
- No server to host/manage
